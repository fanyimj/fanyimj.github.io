java基础面试题

2022-10-16 面试 评论

面向对象与面向过程区别

⾯向对象是⼀种基于⾯向过程的编程思想，是向现实世界模型的⾃然延伸，这是⼀种“万物皆对象”的编程思想。由执⾏者变为指挥者，在现实⽣活中的任何物体都可以归为⼀类事物，⽽每⼀个个体都是⼀类事物的实例。⾯向对象的编程是以对象为中⼼，以消息为驱动。

区别:

- 编程思路不同：⾯向过程以实现功能的函数开发为主，⽽⾯向对象要⾸先抽象出类、属性及其⽅法，然后通过实例化类、执⾏⽅法来完成功能。
- 封装性：都具有封装性，但是⾯向过程是封装的是功能，⽽⾯向对象封装的是数据和功能。
- ⾯向对象具有继承性和多态性，⽽⾯向过程没有继承性和多态性，所以⾯向对象优势很明显

重载和重写的区别

重载：编译时多态、同⼀个类中同名的⽅法具有不同的参数列表、不能根据返回类型进⾏区分

重写（⼜名覆盖）：运⾏时多态、⼦类与⽗类之间、⼦类重写⽗类的⽅法.

方法的重载

方法的重载(overload):

  • 1.定义:在同一个类中允许一个以上的同名方法,只要他们的参数个数或者形参类型不同即可
    • “两同一不同”:同一个类,方法名相同,参数列表不同:参数个数,参数类型不同
  • 2.判断方法是否重载:
    • 跟方法的权限修饰符,返回值类型,形参变量名,方法体都没有关系

方法的重写

  • 方法的重写:子类继承父类以后,可以对父类中的同名同参数的方法,进行覆盖操作

  • 重写后,通过子类对象调用父类中的同名同参数的方法时,实际执行的时子类重写父类的方法

  • 重写的规定:

    • 子类重写的方法的方法名和形参列表与父类被重写的方法的方法名和参数列表相同
    • 子类重写方法的权限修饰符不小于父类被重写的方法(特殊情况:子类不能重写父类中的private权限方)

  • 返回值类型:

    • 父类被重写的方法的返回值类型是void,重写方法也只能是void
    • 父类被重写的方法的返回值类型时是A,重写方法也只能是A及A的子类
    • 父类被重写的方法的返回值类型是基本数据类型,重写方法也只能是相同的基本数据类型

  • 子类重写方法抛出的异常不大于父类重写的方法抛出的异常类型

  • 子类和父类中同名同参数的方法要么都声明为非static的(考虑重写),要么都声明为static(不是重写)

Java 中是否可以重写⼀个 private 或者 static ⽅法？

Java 中 static ⽅法不能被覆盖，因为⽅法覆盖是基于运⾏时动态绑定的，⽽ static ⽅法是编译时静态绑定的。
static ⽅法跟类的任何实例都不相关，所以概念上不适⽤。
Java 中也不可以覆盖 private 的⽅法，因为 private 修饰的变量和⽅法只能在当前类中使⽤， 如果是其他的类继承
当前类是不能访问到 private 变量或⽅法的，当然也不能覆盖。

构造器是否可以被重写？

构造器不能被重写但可以重载

构造⽅法有哪些特性？

  • 名字与类名相同；
  • 没有返回值，但不能⽤ void 声明构造函数；
  • 创建类的对象时⾃动执⾏，⽆需调⽤。
